**Goal:** Replace the binary `nonOverlap` confirmation gate with a continuous ConfirmationScore (0???100) computed from 4 sub-scores on the confirmation candle, with configurable threshold filtering.

**Architecture:** The change is entirely inside `f_htfStrict()` ??? the confirmation block replaces `if nonOverlap` with `if confScore >= _minConfirmScore`. The score flows through to Zone UDT storage, zone labels, and dashboard display. A `minConfirmScore` input (default 0) provides user-configurable filtering ??? 0 preserves V29 behavior.

## DECISION-014

### Continuous ConfirmationScore replaces binary nonOverlap

Decision:

```text
Replace the binary nonOverlap check with a continuous ConfirmationScore
(0-100) computed from 4 weighted sub-scores on the confirmation candle.
```
````

Components:

| Component    | Weight | Formula                                                     |
| ------------ | ------ | ----------------------------------------------------------- |
| Displacement | 40%    | `close - zone_boundary / ATR * 200` (saturates at 0.5x ATR) |
| Rejection    | 30%    | `100 - overlapPct * 100` (0-100 linear)                     |
| FVG          | 20%    | `FVG_gap / ATR * 333` (saturates at 0.3x ATR)               |
| Efficiency   | 10%    | `directional_range / total_range * 100` (0-100 linear)      |

Reason:

The binary nonOverlap gate loses information ??? a confirmation candle that
barely touches the zone tip is treated identically to one that fully engulfs
it. Continuous scoring preserves this nuance and allows users to set their
own quality threshold via `minConfirmScore`.

Default behavior is preserved at `minConfirmScore = 0` ??? all passing
confirmations still create zones.

```markdown
## LESSON-015

### Continuous scoring preserves more information than binary gates

The original `nonOverlap` gate was a boolean: either the confirmation candle
avoids the zone (pass) or doesn't (fail). Replacing it with 4 sub-scores
(Displacement, Rejection, FVG, Efficiency) preserves gradations of quality,
enables user-configurable filtering, and provides richer rejection diagnostics.

Key insight: binary classification gates discard edge-case information.
When a candle barely touches the zone, that's different from full engulfment.
Continuous scoring captures this spectrum.
````
